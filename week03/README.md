# Качествени (нефункционални) изисквания. Сценарии за качество

Видове софтуерни изисквания:
* според нивото:
    * потребителски
    * системни
* според съдържанието:
    * функционални
    * качествени (нефункционални)

## Основни видове качествени характеристики

| Качествена характеристика | Какво показва | Ключови думи |
| --- | --- | --- |
| **Performance (Производителност)** | бързодействието на работа на системата спрямо използваните ресурси | време за отговор, брой заявки, натоварване |
| **Reliability (Надеждност)** | дали системата работи правилно/изправно (т.е. показва вероятността системата да работи без да crash-не) | сривове, откази, възстановяване, работа без прекъсване |
| **Availability (Наличност)** | показва вероятността системата да е налична за работа | наличност, % време работи системата |
| **Security (Сигурност)** | колко е сигурна системата | атаки, неоторизиран достъп, права, защита на данни |
| **Modifiability (Изменяемост)** | колко лесно да се измени системата | промени в системата, време за промяна, странични ефекти |
| **Usability (Използваемост)** | колко е лесна/удобна за употреба системата | лесно използване, обучение на потребителя, грешки от потребителя |
| **Testability (Изпитаемост)** | какъв % от системата може да се тества | тестване, наблюдение на вход/изход, изполиране на компоненти |
| **Fault-tolerance (Отказоустойчивост)** | как/колко добре може да работи системата при наличие на грешки | продължаване на работа при проблем, деградирал режим, устойчивост на грешки |
| **Maintainability (Поддържаемост)** | колко е лесна за поддръжка системата | поддръжка, поправка на дефекти, дългосрочно развитие |
| **Accessibility (Достъпност)** | какъв % от потребителите могат да достъпят системата | достъпност за различни потребители |

## Сценарии за качество
Качествените характеристики се описват чрез **сценарии за качество**

*Сценарий = конкретна ситуация + реакция на системата*

Цел:
* яснота
* формалност
* проверимост

## Структура на сценарии
* **Source (източник)** - *нещото (кой)*, което ни кара да достъпим състоянието, което описваме
* **Stimulus (въздействие)** - *какво прави* нашата система, когато се достига състоянието
* **Artifact (обкет)** - *върху какво* се извършва действието
* **Environment (контекст)** - *при какви условия* се извършва действието
* **Response (реакция)** - *какво се случва* (как системата отговаря на ситуацията)
* **Response Measure (метрика)** - *колко добра* е реакцията (количествена характеристика за резултата от отговора на системата)

*кой -> прави какво -> върху какво -> при какви условия -> какво се случва -> колко добре*
